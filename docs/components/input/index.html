Forms

Ideally, forms are kept simple and short to streamline any actions we want users to take. Single-column forms are preferred where possible; multi-part conditional forms are preferred after that. it is not recommended to use multi-column forms.

The submit button should avoid the word 'submit' by itself where possible, making the action word to be specific to what the user is doing or submitting itself ("submit file"; "save changes"; "calculate results"; etc.).

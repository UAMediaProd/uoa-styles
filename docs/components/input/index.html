Forms and input

Text inputs and text areas

Placeholders can be useful for extra content. It is not recommended to use placeholder elements as a replacement for text labels as placeholders disappear on either text box click event or text entry. We recommend only using placeholders for extra information, examples, or hints for the data entry.

Wrap input and textarea elements with an inputLabel-classed div. Do this for each element on a page or form to enable label highlighting following the focused field.


HTML usage

<div class="inputLabel">
    <label>Text box</label>
    <input type="text"></input>
</div>

<div class="inputLabel">
    <label>Text area</label>
    <textarea rows="8"></textarea>
</div>

Required fields and form validation

Setting the type of inputs and adding an asterisk by applying a required attribute to the input will trigger in-built HTML5 form validation. Adding a class of required to the label for each required field will add the red asterisk to mark required fields.

It is recommended to only use this styling for forms or input fields that contain a mix of optional and required fields. If all fields are required, do not include the required class on the label but keep the input attribute.

Forms

Ideally, forms are kept simple and short to streamline any actions we want users to take. Single-column forms are preferred where possible; multi-part conditional forms are preferred after that. it is not recommended to use multi-column forms.

The submit button should avoid the word 'submit' by itself where possible, making the action word to be specific to what the user is doing or submitting itself ("submit file"; "save changes"; "calculate results"; etc.).

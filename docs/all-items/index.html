Code Samples

To semantically mark text as code in HTML, insert the <code> tag. This displays the text inline, unlike the <pre> tag which has a block display.

You may find the browser default for this tag pretty dull. If, like me, you're using Prism to style code blocks on your site, then you might want your inline <code> blocks to have a similar style to your main <pre> blocks.

This is the style I use, that matches the Okaidia style.


$color-code-background: #272822;
$color-code-color: #f8f8f2;

code {
    background-color: $color-code-background;
    color: $color-code-color;
    border-radius: 0.3rem;
    padding: 4px 5px 5px;
    white-space: nowrap;
}

pre {
  background-color: $color-code-background;
  padding: 5px;
  border-radius: 0.3em;
}

pre code {
  white-space: inherit;
}
  • I've added the white-space rule in to stop text within <code> from splitting on spaces.
  • However, word wrap within <pre> might be broken by this, so you might need to reset (see my CSS pre code { } style)
  • Padding is dependent on the font-size and line-height of your body text - amend as required
  • This is accessible - nice colour contrast <pre>between site background and the text. Use colour contrast checkers like WebAIM's <pre>to verify your colours are accessible.
